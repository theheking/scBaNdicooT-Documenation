Quality Control
===============

This section describes the quality control (QC) steps applied to the scBaNdicooT MMTV-PyMT ALTEN dataset to ensure high-quality cells and reliable downstream analysis.

Overview
--------

Quality control is performed at two levels:

1. **Read-level QC**: Assessed by FastQC and MultiQC reports generated by the Nextflow pipeline.
2. **Cell-level QC**: Applied to the gene-by-cell count matrices after pipeline completion.

Read-Level QC
-------------

The pipeline automatically runs FastQC before and after adapter trimming. Key metrics to inspect:

.. list-table::
   :widths: 30 70
   :header-rows: 1

   * - Metric
     - Expected / Acceptable Range
   * - Per-base sequence quality
     - Phred ≥ 30 for the majority of bases
   * - Adapter content
     - Near zero after trimming
   * - GC content
     - Within expected range for the species (mouse: ~47%)
   * - Duplication rate
     - Variable; moderate duplication expected in scRNA-seq

BD Rhapsody-Specific QC
-----------------------

BD Rhapsody produces paired-end reads where:

* **Read 1** contains the cell barcode and UMI.
* **Read 2** contains the cDNA sequence.

Inspect the following BD Rhapsody-specific metrics in the pipeline summary report:

* **Valid cell barcodes**: Percentage of reads with a valid BD Rhapsody barcode (expected ≥ 85%).
* **Reads per cell**: Median reads per cell (typically 2,000–10,000 for MMTV-PyMT tumoroids).
* **UMI saturation**: A saturation > 50% indicates sufficient sequencing depth.

Cell-Level QC
-------------

After generating count matrices, cell-level QC is performed in R or Python. The following filtering criteria were applied to the MMTV-PyMT dataset:

.. list-table::
   :widths: 30 20 50
   :header-rows: 1

   * - Metric
     - Threshold
     - Rationale
   * - Minimum genes per cell
     - ≥ 200
     - Remove empty droplets / debris
   * - Maximum genes per cell
     - ≤ 6,000
     - Remove multiplets
   * - Minimum UMIs per cell
     - ≥ 500
     - Ensure sufficient coverage
   * - Mitochondrial read fraction
     - ≤ 20%
     - Remove dying / low-quality cells
   * - Nascent UMIs per cell
     - ≥ 50
     - Ensure sufficient labelling efficiency for NTR computation

Example QC Code (Python / Scanpy)
----------------------------------

.. code-block:: python

   import scanpy as sc

   # Load total count matrix
   adata = sc.read_10x_mtx("results/scbandicoot/total/")

   # Calculate mitochondrial gene fraction
   adata.var["mt"] = adata.var_names.str.startswith("mt-")
   sc.pp.calculate_qc_metrics(adata, qc_vars=["mt"], inplace=True)

   # Apply filters
   sc.pp.filter_cells(adata, min_genes=200)
   sc.pp.filter_cells(adata, max_genes=6000)
   sc.pp.filter_genes(adata, min_cells=3)
   adata = adata[adata.obs["pct_counts_mt"] < 20].copy()

   print(f"Cells after QC: {adata.n_obs}")

Labelling Efficiency QC
------------------------

A key QC metric for metabolic labelling experiments is the **labelling efficiency**: the fraction of UMIs classified as nascent genome-wide. This is estimated by the :math:`\hat{\pi}_{1}` parameter of the Beta-mixture model.

Typical values for a 2-hour 4sU pulse in MMTV-PyMT tumoroids:

* **Labelled (Dox + 4sU)**: :math:`\hat{\pi}_{1} \approx 0.15–0.25`
* **Unlabelled control (no 4sU)**: :math:`\hat{\pi}_{1} \approx 0.01–0.03` (background only)

A sample with :math:`\hat{\pi}_{1} < 0.05` in the 4sU-treated condition may indicate poor labelling efficiency and should be flagged for further investigation.

Doublet Detection
-----------------

Doublets (two cells captured in one droplet) are detected using `Scrublet <https://github.com/swolock/scrublet>`_ or `DoubletFinder <https://github.com/chris-mcginnis-ucsf/DoubletFinder>`_:

.. code-block:: python

   import scrublet as scr

   scrub = scr.Scrublet(adata.X, expected_doublet_rate=0.06)
   adata.obs["doublet_score"], adata.obs["predicted_doublet"] = scrub.scrub_doublets()

   # Remove predicted doublets
   adata = adata[~adata.obs["predicted_doublet"]].copy()
